# Trustworthy IoT Security Analytics: Bridging Symbolic Reasoning and Generative AI

[![Accepted in EMSE](https://img.shields.io/badge/Status-Accepted_in_EMSE-brightgreen.svg)]()
[![Lab](https://img.shields.io/badge/Lab-CRESSET-blue.svg)]()

Welcome to the official repository for **StaticAnalysis_vs_LLM**, containing the evaluation framework, datasets, and hybrid pipeline developed by the CRESSET Lab at Toronto Metropolitan University. This work has been accepted for publication in the Empirical Software Engineering (EMSE) journal.

## 📖 Overview

As smart home ecosystems and IoT environments become more complex, independent automation rules can inadvertently collide, creating Rule Interaction Threats (RITs) that pose serious safety risks. This project investigates the limitations of both traditional Static Analysis and modern Large Language Models (LLMs) in detecting these threats, and introduces a novel **Neuro-Symbolic Hybrid Architecture** to reconcile the two. 

## ⚠️ The Challenge

Rule Interaction Threats (RITs) are generally classified into Action Contradictions (AC), Trigger Cascades (TC), and Condition Cascades (CC). Detecting these requires an understanding of both rigid code structure and human intent. 

*   **The Incumbent (Static Analysis - oHIT):** Symbolic execution frameworks offer near 100% recall, finding every theoretical overlap. However, they suffer from context blindness, resulting in high false-positive rates and alert fatigue (e.g., flagging a conflict between "8:00 AM" and "Sunset").
*   **The Challenger (Generative AI):** While LLMs excel at semantic intuition and understanding human intent, our study shows they are structurally blind. When tested against complex logic chains, standard models experience a "generalization collapse" and hallucinate heavily. 

## 🛠️ Our Solution: The Reconciliation Pipeline

To bridge this gap, we designed an LLM-oHIT Reconciliation Architecture. 

We do not ask the LLM to search the code for vulnerabilities; instead, we hand it the "haystack" of alerts generated by the high-recall static analysis tool. The LLM acts purely as a semantic adjudicator, performing a "common sense" check to filter out benign, intentional interactions from actual threats. 

## 📊 Datasets and Stress Testing

To rigorously evaluate the models without relying solely on limited real-world examples, we developed a comprehensive evaluation suite:
*   **Real-World Benchmark (oHC/IoTB):** A dataset of 145 manually verified rule interactions sourced from the openHAB community.
*   **The Mutation Framework:** We created a robust stress-test dataset consisting of 2,495 unique, synthetically mutated rulesets. Using a TXL mutation engine, we injected specific RITs to systematically break model pattern-matching and evaluate true reasoning. 

## 🚀 Key Results

*   **Massive Precision Boost:** The hybrid pipeline increased detection precision by 20 points, from the 73% baseline of static analysis to 93%, successfully eliminating noise and alert fatigue.
*   **100% Blind Spot Recovery:** The semantic intuition of the LLM successfully recovered 100% of the "Strong Condition Cascades" that the static parsing tool missed due to rigid syntax errors. 
*   **Adversarial Robustness:** Under the pressure of the 2,495 mutated rulesets, standard models (like Llama-8B and Llama-70B) completely collapsed, failing on structure-dependent strong threats. Only alignment-focused reasoning models (such as GPT-4o and Gemini 2.5 Pro) maintained logic despite code mutation.

## 💡 Key Takeaway
*"Security is no longer just about code correctness; it is about semantic intent."* 
Neural networks are probabilistic, whereas symbolic logic provides the safety guarantees required for critical infrastructure. The future of safety-critical systems is a neuro-symbolic hybrid.
