Now that you have classified the rule interactions as an AC. You have been tasked with classifying the rules as Weak Action Contradiction (WAC) or a Strong Action Contradiction (SAC) 

RULE STRUCTURE
Rules follow this format:
rule "Name"
when
    [TRIGGER]
then
    [CONDITIONS & ACTIONS]
end

Key Components:
- Triggers: Found between "when" and "then"
- Actions: Commands using sendCommand or postUpdate
- Conditions: If-statements guarding actions, variables CHANGED to a value 
".state" after a variable simply checks the value of the variable 

THREAT CLASSIFICATIONS AND PATTERNS

1. Strong Action Contradiction (SAC)

To identify a SAC, 3 conditions must be satisfied:
Feature 1: Overlapping triggers
Feature 2: NO conditions
Feature 3: Contradictory actions

Example:
rule "Fire Emergency"
when
	Item Smoke_Detector changed to ON
then
	sendCommand(Window_Lock, OFF)
	sendCommand(Door_Lock, OFF)
sendCommand(Lights, ON)
end

rule "Bedtime"
when
	Time cron "0 0 23 * * ?" // Triggered daily at 11 PM
then
	sendCommand(Window_Lock, ON)
	sendCommand(Door_Lock, ON)
	sendCommand(Lights, OFF)
end

2. Weak Action Contradiction (WAC)

Feature 1: Overlapping triggers
Feature 2: Overlapping conditions. Multiple conditions are allowed in either rules. 
Feature 3: Contradictory actions

Example 1:

rule "Carbon Monoxide Alert"
when
	Item CO_Detector changes to ON
then
	sendCommand(Windows, OPEN)
	sendCommand(Fans, ON)
end

rule "Potential Security Threat Detected"
when
	Item OutdoorSecurityCamera_Motion changed to ON
then
	if (Security_Mode == ON) {
	sendCommand(Windows, CLOSE)
	sendCommand(DoorLock, ON)
	}
end

The difference between a WAC and a SAC is that a SAC has NO CONDITIONS in each rule. 
A WAC has conditions in at least one of its rules.
