Now that you have classified the rule interactions as a CC. You have been tasked with classifying the rules as Weak Condition Cascade (WCC) or a Strong Condition Cascade (SCC)

RULE STRUCTURE
Rules follow this format:
rule "Name"
when
    [TRIGGER]
then
    [CONDITIONS & ACTIONS]
end

Key Components:
- Triggers: Found between "when" and "then"
- Actions: Commands using sendCommand or postUpdate
- Conditions: If-statements guarding actions, variables CHANGED to a value 
".state" after a variable simply checks the value of the variable 

THREAT CLASSIFICATIONS AND PATTERNS

1. Strong Condition Cascade (SCC)
To identify a SCC, 2 conditions must be satisfied:
Feature 1: Overlapping triggers
Feature 2: All-Condition enabling action

Example 1:
rule "Initialize Lock Mode for Barn Door"
when
    System started
then
    sendCommand(BarnDoor_LockMode, 1)
end

rule "Open Barn Doors for Feeding"
when
    Time cron "0 0 8 * * ?" // Trigger at 8 AM daily
then
    if (BarnDoor_LockMode.state == 1)
        sendCommand(BarnDoor, OPEN)
end

2. Weak Condition Cascade (WCC)
To identify a WCC, 2 conditions must be satisfied:
Feature 1: Overlapping triggers
Feature 2: Some-Condition enabling action

Example 1:  
rule "Initialize Lock Mode for Barn Door"
when
    System started
then
    sendCommand(BarnDoor_LockMode, 1)
end

rule "Open Barn Doors for Feeding"
when
    Time cron "0 0 8 * * ?" // Trigger at 8 AM daily
then
    if (BarnDoor_LockMode.state == 1)
        if (ExtremeWeatherWarning.state == OFF)
            sendCommand(BarnDoor, OPEN)
end

The difference between a WCC and a SCC: 
- SCC has multiple conditions, the first condition is enabled due to an action in the first rule. 
- WCC has ONE condition.
